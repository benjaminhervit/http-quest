GAME MANUAL

This game is about learning the basics of HTTP requests. Each level will give you a setting and a small (re)quest that you must execute sucessfully to progress. Each time you complete a section, your xp will increase, which is shown on the leaderboard.

RULES

  • All quests (with few exceptions) are from domain.com/game/quest-slug-goes-here
  • Each quest only accept specific methods: GET, POST, PUT, and/or DELETE. All the information you need should be in the quest description.
  • A quest only accepts one data format: domain/game/quest-slug/<parameters>, { JSON data }, or Form data
  • When registered, always include your registered username in the authorization headers.

HOW TO COMPLETE A reQUEST
Your request must be the correct combination of:

  • url/path/paramter(s): url = domain.com path=/game/some-quest-slug/ parameter=maybe-a-parameter-here
  • method: GET, POST, PUT or DELETE
  • data format: parameters, JSON or Form
  • quest answer: often some bad pun or word from the quest.
  • authorization: include your username in the authorization header when you

HOW TO WIN

  • To win the game, you must complete all quests. You can follow your progress on the leaderboard.

    • HINTS

    • Most of the game can easily be played using a HTTP CLIENT - but it is also a good chance to practice some python and JavaScript ir you want to make it harder for yourself.
    • You should always receive a JSON response from the game - unless something broke or you are outside the "playingfield" (that means we could not recognize your url).
    • If your request fails, read the error message carefully. It should give you hints on where the error is - or if you should tell us that we have made an error in the game.
    • Read the quest description carefully. They are usually a bad pun on some basic request paramters.
    • /log/last_request/username can be a helpful debugging tool when you start making requests with authorization.
